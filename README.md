# DVerseClaudeSkills

> **What if your AI coding assistant already knew Dynamics 365 CE — the event pipeline, the best practices, the exact SDK patterns — before you typed a single word?**

This repo demonstrates exactly that. It pairs **Claude Code** (Anthropic's AI coding agent) with a custom **domain skill** that encodes senior-level Microsoft Dataverse architecture knowledge. The result: production-quality Dataverse backend components — plug-ins, workflow extensions, unit tests, and documentation — generated from a plain English description.

---

## The Idea

Most developers use AI tools as smart autocomplete. Claude Code goes further — it's an autonomous agent that reads files, writes code, runs commands, and scaffolds entire project structures. But out of the box, it doesn't know the Dynamics 365 CE event pipeline, or that you should never call the Web API inside a plug-in, or that `FilteringAttributes` must be set on Update steps.

That's where **Claude Code Skills** come in.

A skill is a structured prompt file you drop into Claude Code's skills folder. When invoked, it transforms Claude Code into a specialist — in this case, a senior D365 CE solution architect who:

- Grounds every decision in the official **Microsoft Dynamics 365 CE Developer Guide v9.1** and **Dataverse Developer docs**
- Follows all 12 official plug-in best practices by default
- Writes code directly to disk — `.cs` files, `.csproj` files, test projects, docs — without being asked
- Remembers your project conventions (publisher prefix, solution name) across sessions

## How the Skill Works

Claude Code skills follow a simple contract:

1. **`SKILL.md`** — the entry point. Defines the agent's persona, operating mode, response pattern, and what to avoid. The frontmatter block (`name`, `description`, `triggers`) tells Claude Code when to auto-activate it.

2. **`references/`** — domain knowledge files. The skill instructs Claude to read the relevant file before answering. Each file is under 300 lines and covers one domain: plug-in development, data access, security, ALM, integration, or bootstrap. This keeps the knowledge current and auditable — you can read and edit it yourself.

When you type `/d365-architect` in Claude Code, the skill loads and Claude becomes a D365 specialist for that session. It knows the pipeline stages, the SDK patterns, the ALM commands. You describe a business requirement; it writes the code.

## What the Skill Generated

Two fully scaffolded Dataverse backend components, each produced from a single plain-English request:

### BlockOrderCreateOnCreditHold
A **PreValidation plug-in** on `dexx_order` Create. Before the order is saved, it checks the related Account's `creditonhold` field. If true, it throws an `InvalidPluginExecutionException` — the operation is cancelled before it ever hits the database transaction. Includes an isolated `.csproj`, strong-name signing reference, and a full xUnit + Moq test suite covering guard clauses, missing lookups, happy path, and exception message content.

### CreateOpportunityOnStatusChange
A **custom workflow activity** (`CodeActivity`) on Account Update. When an Account's status code changes to a specific value, it creates an Opportunity and a 15-day follow-up Task, resets the Account status, and returns `OutSuccess` and `OutOpportunityId` to the workflow designer — making the outcomes available as tokens in the Power Automate / classic workflow canvas. Same test and documentation structure.

## Getting Started

**1. Install Claude Code**
```bash
npm install -g @anthropic-ai/claude-code
claude
```

**2. Clone this repo and install the skill**

Windows:
```powershell
git clone https://github.com/tbalt88/DVerseClaudeSkills.git
Copy-Item -Recurse "DVerseClaudeSkills\d365-architect" "$env:USERPROFILE\.claude\skills\d365-architect"
```

Mac / Linux:
```bash
git clone https://github.com/tbalt88/DVerseClaudeSkills.git
cp -r DVerseClaudeSkills/d365-architect ~/.claude/skills/d365-architect
```

**3. Invoke the skill and set your project context**
```
/d365-architect
prefix = yourprefix, solution = YourSolution
```

Claude Code saves your conventions to persistent memory. From that point forward, every plug-in it generates uses your prefix, your solution name, and your folder structure — automatically.

**4. Describe a requirement**
```
Write a PreValidation plugin on dexx_invoice Create that blocks
creation if the related contract is expired.
```

Claude Code will write the `.cs` file, the `.csproj`, the test project, and the documentation to disk.

## Why This Matters

The skill pattern solves a real problem with AI-assisted development on enterprise platforms: **generic AI tools don't know your platform's constraints**.

Dynamics 365 CE has specific rules that matter:
- You cannot call the Web API inside a plug-in
- Cancelling in PreOperation causes a rollback — you must cancel in PreValidation
- Every Update step registration needs `FilteringAttributes` or the plug-in fires on every field change
- Plug-in classes must be stateless — no instance fields

A generic AI will get these wrong. A skilled AI won't, because the rules are encoded in the skill definition and reference files — and you can inspect, update, and own those files.

This is the model: **domain knowledge as code, checked into source control, versioned alongside the artifacts it produces.**
